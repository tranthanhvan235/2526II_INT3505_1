# So sánh OpenAPI, API Blueprint, RAML, TypeSpec
## Bảng so sánh
| Tiêu chí    | OpenAPI    | API Blueprint | RAML       | TypeSpec        |
| ----------- | ---------- | ------------- | ---------- | --------------- |
| Format      | YAML/JSON  | Markdown      | YAML       | DSL riêng       |
| Độ phổ biến | Rất cao    | Trung bình    | Khá        | Mới             |
| Tooling     | Rất mạnh   | Hạn chế       | Khá tốt    | Đang phát triển |
| Dễ học      | Trung bình | Dễ            | Trung bình | Khó hơn         |
| Sinh code   | Rất tốt    | Kém           | Tốt        | Rất tốt         |
| Testing     | Tốt        | Hạn chế       | Tốt        | Tốt             |

### Nhận xét

**OpenAPI**: chuẩn phổ biến nhất, hệ sinh thái mạnh (Swagger, Postman, codegen).

**API Blueprint**: dễ viết nhất, nhưng ít tool hỗ trợ.

**RAML**: cú pháp sạch, có tái sử dụng tốt, nhưng ít phổ biến hơn OpenAPI.

**TypeSpec**: viết API như code, phù hợp hệ thống lớn, nhưng còn mới.
